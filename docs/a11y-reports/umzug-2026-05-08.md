## Issues to fix (in priority order)

**No P0/P1 (blocker) issues.** The items below are advisory polish for WCAG 2.2 AA and BITV 2.0 future work.

1. **P3 — Touch target size on default/small buttons** (advisory; WCAG 2.5.5 AAA / WCAG 2.2 AA 2.5.8). `src/components/ui/button.tsx:24,26,28,32` defines `default = h-8` (32px), `sm = h-7` (28px), `xs = h-6` (24px), `icon = size-8` (32px), `icon-sm = size-7` (28px). All meet the 24px WCAG 2.2 AA floor; none meet the 44px AAA target. Cascade impact: every CTA button on Preview/Run, plus the Topbar `UserMenu`/`ThemeToggle`/`LanguageSwitcher`. **Recommendation**: bump default button height to `h-10` (40px) or `h-11` (44px) before the public Loom recording, especially for primary CTAs `"Autopilot starten"` and `"Mit eID bestätigen"`. Not a PASS blocker for WCAG 2.1 AA.

2. **P3 — `<details>` summary semantics for the Prototype Disclaimer** (BITV 2.0 readability, not a WCAG violation). `src/components/shared/PrototypeDisclaimer.tsx:16–28` uses native `<details>/<summary>` which is the correct pattern. The role file's BITV 2.0 advice asks for a "plain language toggle (Leichte Sprache) for at least the dashboard + autopilot results" — this is **not** present in the current build; the prototype disclaimer is a different artefact. Mark as future work (see BITV section below).

3. **P3 — Footer landmark uniqueness** (advisory). `(app)/layout.tsx` renders `<Footer>` which itself contains `<footer>` + a nested `<nav aria-label="Footer">`. The nested footer-nav uses `aria-label="Footer"` which is hardcoded and not translated. WCAG 3.1.2 only mandates lang attribution, not translation, so this is fine for AA, but the footer nav label should be moved to an i18n key for consistency with the rest of the app — see `src/components/layout/Footer.tsx:15`.

4. **P3 — `data-bussgeld-context` audit attribute is correctly isolated**. Audited as expected by spec §11; `FristDetailModal.tsx:46` is the sole carrier of `data-bussgeld-context="frist-detail-modal"` and the sole renderer of `common.disclaimer.bussgeld_frist_modal`. No issue.

5. **P3 — Skip-link wording**. The skip link uses `tApp('skip_to_content')` which exists in `de.json`; however the empty stub locale files (`en/ru/uk/ar/tr.json` are `{}` per frontend-coder log) mean fallback is to the key itself in non-DE renders. Since i18n-localizer is the next pipeline step, this is expected; flag for that agent rather than as an a11y blocker.
